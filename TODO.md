## 详细任务清单

### Task 01：创建 Node.js 全栈 Monorepo 脚手架

状态：`[x] 已完成`

目标：建立 MVP 的基础工程结构，让 Web、共享包、数据库包、后台 Worker 和部署配置可以在同一个仓库内协同开发。

实现内容：
- 使用 pnpm workspace 管理 `apps/*`、`packages/*`、`workers/*`。
- 使用 Turborepo 串联 `dev`、`build`、`lint`、`typecheck` 等基础命令。
- 保留 `apps/web` 作为 Next.js Web 与 API 入口。
- 保留 `packages/database`、`packages/shared`、`packages/ui`、`packages/scanner`、`packages/reader-core` 作为 MVP 共享包边界。
- 使用 Python watchdog worker 作为扫描任务 Worker 入口。

验收标准：
- 根目录可以通过 `pnpm install` 安装依赖。
- 根目录可以通过 `pnpm dev` 启动开发服务。
- 根目录可以通过 `pnpm build` 执行构建任务。
- `apps/web/app/api/health/route.ts` 可返回健康检查结果。

### Task 02：迁移高保真原型为 Next.js 页面

状态：`[x] 已完成`

目标：把 MVP 高保真原型迁移为可运行的 Next.js 页面，先完成主要信息架构和交互外观。

实现内容：
- 使用 Next.js App Router 建立首页、书库、书架、整理、扫描任务、设置、登录、阅读器、移动端预览等页面。
- 将原型拆分为页面级 feature 组件和可复用 UI 组件。
- 使用 Tailwind CSS 与主题样式承接视觉规范。
- 当前阶段可使用 mock 数据，但页面结构需要为后续接真实 API 预留清晰边界。

验收标准：
- 主要页面路由可直接访问且无运行时错误。
- 页面组件不依赖外部原型工具即可渲染。
- 导航入口能覆盖 MVP 主要页面。
- mock 数据集中维护，后续可以替换为 API 数据。

### Task 03：设计 Prisma MVP 数据模型

状态：`[x] 已完成`

目标：设计支撑书库、文件、扫描任务、阅读进度和基础认证的 MVP 数据模型。

实现内容：
- 扩展当前极简 `LibraryItem` 占位模型。
- 增加用户、会话或认证凭据相关模型。
- 增加书库路径配置模型，记录 NAS 根路径、启用状态和扫描策略。
- 增加读物、文件、章节或资源条目模型，支持 TXT、PDF、图片 / 漫画等格式。
- 增加扫描任务和扫描日志模型，记录状态、进度、错误和耗时。
- 增加阅读进度模型，记录用户、读物、当前位置、百分比和更新时间。

验收标准：
- Prisma schema 能覆盖 MVP 的核心实体与关系。
- 能执行 Prisma generate。
- 模型字段足够支持 Task 04 到 Task 14 的 API 与页面需求。
- 不引入超出 MVP 的复杂权限、多租户或推荐系统模型。

### Task 04：实现基础认证

状态：`[x] 已完成`

目标：提供单用户或轻量多用户的基础登录能力，保护管理页面、阅读页面和文件访问 API。

实现内容：
- 实现登录页表单和登录 API。
- 实现密码校验、会话创建、会话续期和退出登录。
- 增加受保护路由逻辑，未登录用户跳转到登录页。
- 增加 API 鉴权中间件或统一 helper，保护书库、扫描、阅读进度和文件访问接口。
- 为本地部署提供初始管理员账号配置方式。

验收标准：
- 未登录访问受保护页面会被拦截。
- 登录成功后可以访问书库和阅读页面。
- 退出登录后会话失效。
- 未认证请求无法访问受控文件 API。

### Task 05：实现书库路径配置

状态：`[x] 已完成`

目标：允许用户配置 NAS 上的读物根目录，并为后续扫描任务提供稳定输入。

实现内容：
- 在设置页增加书库路径配置表单。
- 实现书库路径的新增、编辑、启用 / 禁用和删除。
- 在服务端校验路径格式、路径是否存在、是否可读。
- 将书库路径配置持久化到数据库。
- 提供 API 给扫描任务选择目标书库路径。

验收标准：
- 用户可以在页面上保存至少一个书库根路径。
- 不存在或不可读路径会给出明确错误。
- 禁用的路径不会进入扫描任务。
- 已保存路径刷新页面后仍然存在。

### Task 06：实现基础书库 API

状态：`[x] 已完成`

目标：提供读物数据的基础增删改查能力，作为列表、详情、阅读器和扫描入库的统一数据来源。

实现内容：
- 实现读物列表 API，支持分页、搜索、格式筛选和基础排序。
- 实现读物详情 API，返回元数据、文件信息、章节 / 页信息和阅读进度摘要。
- 实现读物元数据更新 API，支持标题、作者、封面、标签或描述等 MVP 字段。
- 实现读物删除或隐藏 API，避免误删 NAS 源文件。
- 定义统一响应格式和错误处理方式。

验收标准：
- 前端可以通过 API 获取读物列表和详情。
- 搜索、分页和格式筛选结果稳定。
- 更新元数据后列表与详情能看到最新值。
- 删除或隐藏操作不会直接破坏源文件。

### Task 07：实现 NAS 目录扫描 Worker

状态：`[x] 已完成`

目标：实现后台扫描 Worker，从配置的 NAS 目录中发现读物文件并写入数据库。

实现内容：
- 基于 BullMQ 消费扫描任务队列。
- 在 `packages/scanner` 中实现目录遍历、文件类型识别和基础元数据提取。
- 支持 TXT、PDF、图片目录、漫画压缩包或图片集合的 MVP 文件识别。
- 将扫描结果 upsert 到数据库，避免重复入库。
- 记录扫描进度、发现数量、跳过数量、错误日志和完成状态。

验收标准：
- Worker 可以消费扫描任务并输出真实扫描结果。
- 重复扫描同一目录不会产生重复读物。
- 文件不可读或格式不支持时记录错误但不中断整个任务。
- 扫描完成后书库 API 能查到新增读物。

### Task 08：实现扫描任务 API 和页面

状态：`[x] 已完成`

目标：提供创建、查看和追踪扫描任务的 API 与页面，连接 Web 和后台 Worker。

实现内容：
- 实现创建扫描任务 API，接收书库路径配置 ID。
- 实现扫描任务列表和详情 API，返回状态、进度、日志和错误。
- 页面接入真实 API，展示任务队列、运行中任务、完成任务和失败任务。
- 支持手动触发扫描和刷新任务状态。
- 对重复触发、路径不存在、Worker 不可用等情况给出明确反馈。

验收标准：
- 用户可以从页面手动创建扫描任务。
- 任务创建后 Worker 能消费并更新状态。
- 页面能看到任务进度、完成结果和失败原因。
- 刷新页面后任务历史仍然存在。

### Task 09：实现读物列表和详情页接入真实 API

状态：`[x] 已完成`

目标：用真实数据库和 API 替换当前 mock 数据，让书库浏览进入可用状态。

实现内容：
- 列表页调用读物列表 API，处理加载、空状态、错误状态和分页状态。
- 详情页调用读物详情 API，展示真实元数据、文件信息和阅读入口。
- 保留现有视觉结构，替换数据来源和交互逻辑。
- 在读物缺失或无权限时展示明确错误。

验收标准：
- 扫描入库后的读物会出现在列表页。
- 点击读物可进入真实详情页。
- 列表和详情不再依赖 mock 数据作为主数据源。
- API 失败时页面不会崩溃。

### Task 10：实现文件受控访问 API

状态：`[x] 已完成`

目标：所有读物源文件必须通过受控 API 访问，避免直接暴露 NAS 真实路径。

实现内容：
- 实现文件访问 API，根据读物或文件 ID 定位源文件。
- 做认证校验和读物权限校验。
- 做路径归一化和根目录约束，防止路径穿越。
- 支持 `Range` 请求和流式返回，满足 PDF、图片和大文本阅读。
- 设置合适的 Content-Type、Content-Disposition、缓存和错误响应。

验收标准：
- 未登录用户无法读取文件。
- 请求无法越过已配置书库根目录。
- 大文件可以分段或流式读取。
- PDF、图片和 TXT 阅读器都通过该 API 获取内容。

### Task 11：实现 TXT 阅读器

状态：`[x] 已完成`

实现内容：
- 通过受控文件 API 加载 TXT 内容。
- 支持编码识别或 MVP 默认 UTF-8，并对异常编码给出提示。
- 实现阅读区域、字体大小、行高、主题和滚动位置控制。
- 将当前位置转换为可保存的进度数据。
- 进入阅读器时恢复上次阅读位置。

验收标准：
- TXT 文件可以正常打开和滚动阅读。
- 刷新或重新进入后能恢复到上次位置。
- 字体大小、行高或主题设置至少支持 MVP 级调整。
- 大文本加载失败时有明确错误提示。

### Task 12：实现 PDF 阅读器

验收标准：
- PDF 文件可以在阅读器中打开。
- 用户可以翻页和缩放。
- 关闭后再次打开能恢复到上次页码。
- PDF 文件不会暴露 NAS 真实路径。

验收标准：

- 图片加载失败不会导致整个阅读器崩溃。
- 重新进入后能恢复到上次页码。
- 所有图片访问都经过受控文件 API。

### Task 14：实现阅读进度保存与恢复

状态：`[x] 已完成`

目标：为 TXT、PDF、图片 / 漫画阅读器提供统一的阅读进度保存与恢复能力。

实现内容：
- 在 Prisma 中增加阅读进度模型。
- 实现读取进度 API，按用户和读物返回最新进度。

- 在阅读器中做节流保存，避免高频请求。
- 进入读物详情和阅读器时展示或恢复最近进度。
- MVP 阶段采用最后写入优先的多设备基础策略。

验收标准：
- 三类阅读器都能保存并恢复进度。
- 同一用户不同读物的进度互不影响。
- 刷新页面或重新登录后进度仍然存在。
- 高频滚动或翻页不会产生过量 API 请求。
